# Moving this app to another PC

This is a Next.js + SQLite app. It's fully portable — you just **rebuild the
dependencies on the new PC** (the database driver is compiled per-machine, so
`node_modules` can't be copied) and copy your data file if you want your
campaigns to come along.

## Prerequisite on the new PC
Install **Node.js 20 or newer** from https://nodejs.org (the LTS installer is
fine). That's the only prerequisite.

---

## Method A — Copy the folder (simplest, brings your campaigns)

1. On THIS PC, copy the project folder
   `D:\_______Claude\aa-anniversary-companion`
   to a USB drive / network share / cloud folder — **but skip these** (they're
   huge and/or machine-specific; they get rebuilt):
   - `node_modules\`  (rebuilt by setup)
   - `.next\`  (build cache)
   - `src\generated\prisma\`  (regenerated by setup)
   - the `*.png` screenshots and `dev-server.log` (optional, not needed)

   **Keep `dev.db`** if you want your existing campaigns on the new PC.
   **Keep `.env`** to bring your API key (or recreate it — see step 3).

2. On the NEW PC, put the folder anywhere (e.g. `C:\war-ledger`), open
   **PowerShell** in that folder, and run:
   ```
   powershell -ExecutionPolicy Bypass -File .\setup.ps1
   ```
   This installs dependencies, rebuilds the database driver, regenerates the
   Prisma client, and prepares the database.

3. If you didn't copy `.env`, the setup created one from the template — open
   `.env` and paste your `ANTHROPIC_API_KEY` (optional; only the AI assistant
   tab needs it).

4. Start it:
   ```
   powershell -ExecutionPolicy Bypass -File .\start.ps1
   ```
   It launches the server hidden and opens http://localhost:3000. (Or just
   `npm run dev` if you prefer a visible terminal.)

---

## Method B — Git (best if you'll keep developing on both PCs)

On THIS PC (one-time):
```
cd D:\_______Claude\aa-anniversary-companion
git init
git add .
git commit -m "War Ledger"
```
Push to a **private** GitHub repo (`.env` and `node_modules` are already
git-ignored, so your API key is NOT committed):
```
git remote add origin <your-private-repo-url>
git push -u origin main
```

On the NEW PC:
```
git clone <your-private-repo-url> war-ledger
cd war-ledger
powershell -ExecutionPolicy Bypass -File .\setup.ps1
```
Then create `.env` (copy `.env.example` → `.env`, paste your key) and run
`start.ps1`. Note: git does **not** carry `dev.db` (it's your data) — copy that
file over separately if you want your existing campaigns; otherwise you start
with an empty, fresh database.

---

## Notes
- **Not Windows?** `setup.ps1`/`start.ps1` are PowerShell helpers. On macOS/Linux
  the equivalent is: `npm install` → `npm run setup` → `npm run dev`. The app
  itself is cross-platform; only the hidden-window restart helper is Windows-tuned
  (it has a plain fallback elsewhere).
- **Updating the new PC later** (Method B): `git pull` then `npm install` (its
  `postinstall` regenerates the Prisma client) and, if the schema changed,
  `npm run setup`.
- **Your API key** travels in `.env`. Treat that file as a secret — don't commit
  it or share the folder publicly with it inside.
